# 00-00-idea-brainstorm

- REVIEWER_MODEL = `gpt-5.5` — Model used via Codex MCP.
- MAX_POST_REVIEW_ROUNDS = 10 — Post-review 迭代轮数上限。

围绕用户给定的研究方向、约束与偏好，生成 `00-00-idea-pool.md`。

## 输入

- 用户描述的研究方向/主题
- 可选约束：
  - 算力/时间预算
  - 偏计算 / 偏理论 / 偏数值模拟（或偏实验）
  - 目标 venue
  - 已有数据、代码、实验资产

## 输出

`00-00-idea-pool.md` -- 主要语言用中文，专业术语保留英文。

## 写作风格（重要）

生成的 idea pool 必须以**连贯的叙事段落**呈现，而非填表式分条列举。核心要求：

- **逻辑连贯，叙事成段**：每个 idea 用 2-4 段连贯叙述将问题、动机、洞见、方法、风险串联起来。段落之间要有逻辑递进关系——不是把几个维度分别写一句话然后拼在一起，而是让读者能顺着一条推理链读下去。
- **展示推理过程而非结论列表**：写出为什么某个方向值得探索、核心假设是什么、哪些环节存在不确定性、优先验证什么。推理链应当完整且可追溯——读者应能理解每一步判断的依据。
- **精确而非空泛**：避免"该方向具有重要的研究价值""值得进一步探索"等无信息量的表述。每个判断必须有具体依据。如果说不清为什么重要，就不要写"重要"。
- **信息密度优先**：如果一个维度在某个 idea 上无实质内容可写，就省略。idea 描述的篇幅由其复杂度决定，不要求均匀分布。

## 工作流

### Step 1: Pre-review

调用 `mcp__codex__codex` 检查生成计划：

```
mcp__codex__codex:
  model: gpt-5.5
  prompt: |
    请检查以下 idea brainstorming 计划是否合理：

    研究方向: {用户描述的研究方向}
    约束条件: {算力/时间/偏好等约束}
    执行计划: 生成 6-12 个候选 idea，每个以自然叙事段落呈现，强调思考过程和推理链

    检查：是否覆盖上游要求？是否自洽？有无遗漏或过度扩展？
```

### Step 2: 生成候选 idea 池

生成 6-12 个候选 idea。每个 idea 按以下**松散格式**输出——这不是模板，是思考方向，实际写作时用叙事段落把这些内容串起来：

```
### Idea Ixx: [简短标题，直接用中文写]

[用 2-4 段叙事把以下内容串起来，不要分条列举：]

- 要解决什么问题？为什么现在值得做？（不要写"填补研究空白"这种话——说清楚具体动机）
- 核心洞见是什么？这个洞见从哪来的？（是来自某篇 paper 的启发、某个现象的观察、还是理论推导的自然延伸？）
- 方法上怎么做？最小可行路径是什么？（不需要完整方法论，但要写出关键设计选择）
- 理论可分析的部分在哪？能用数值模拟验证什么？（具体到 setting / control variables / expected observable）
- 如果成功，别人会记住什么？如果失败，最可能卡在哪？
- 适合投什么 venue？为什么？
```

关键约束：
- 候选之间要有**实质差异**，不能只是换措辞
- 默认偏向"计算/理论/数值模拟"方向——优先选能形成 theory → simulation 闭环的 idea
- 每个 idea 必须能落到**最小验证实验**（优先数值模拟可验证）
- 明显依赖大规模数据采集、重工程实现、且缺乏理论增量的方向，直接降权并写明原因
- 对每个 idea，**明确写出你最大的不确定性**——哪个假设最需要先验证

### Step 3: Post-review（迭代循环，最多 10 轮）

调用 `mcp__codex__codex` 检查 idea pool 质量：

```
mcp__codex__codex:
  model: gpt-5.5
  prompt: |
    请检查以下 idea pool 是否适合作为论文前置 brainstorming 输出：

    {idea pool 内容}

    检查：是否覆盖上游要求？是否自洽？有无遗漏或过度扩展？
    额外检查：叙事是否逻辑连贯？推理链是否完整可追溯？还是读起来像填表？

    若有问题，明确指出并给出修改建议。
```

迭代逻辑：
- 若 review 指出问题 → 按 review 建议修改 idea pool → 继续 review（round++）
- 若 review 通过或达到轮数上限 → 结束

**每轮情况汇总**：review 循环结束后，打印每轮的简要情况。

### Step 4: 输出确认

输出：
- `00-00-idea-pool.md` 已生成
- 用一段话总结最值得推进的 3-5 个方向，以及为什么它们脱颖而出
- 提示下一步：`/00-01-idea-evaluate`
